# Docker 이미지 자동배포 (GitHub Actions + Self-hosted Runner) 구축 가이드

## 1. 이미지 빌드 전략 — 소스는 이미지에 COPY

`docker-compose.yml`에서 소스 코드 볼륨 마운트는 제거하고, Dockerfile에서 COPY로 완전히 포함시킨다.

```dockerfile
# 의존성 설치는 위쪽 (캐시 재사용 목적)
COPY requirements.txt .
RUN pip install -r requirements.txt

# 소스는 맨 아래 (자주 바뀌는 부분)
COPY ./backend /workspace/backend
```

```yaml
# docker-compose.yml — 소스 볼륨 마운트 없음
volumes:
  # - ./backend:/workspace/backend   ← 반드시 제거/주석 처리
  - ./ai_model:/workspace/ai_model    # 대용량 모델 파일만 볼륨 유지
```

## 2. 이미지 태깅 — git commit hash 기반(개념설명)

```bash
TAG=$(git rev-parse --short HEAD)
docker build -t {레지스트리주소}/{이미지명}:$TAG \
             -t {레지스트리주소}/{이미지명}:latest .
docker push {레지스트리주소}/{이미지명}:$TAG
docker push {레지스트리주소}/{이미지명}:latest
```

## 3. Self-hosted Runner 설치

```bash
mkdir -p ~/actions-runner/{레포명} && cd ~/actions-runner/{레포명}

curl -o actions-runner-linux-x64.tar.gz -L {GitHub이_안내하는_다운로드_URL}
tar xzf actions-runner-linux-x64.tar.gz

./config.sh --url https://github.com/{owner}/{repo} --token {발급받은_토큰}
# Runner 이름, labels 지정 (예: gpu,docker,wsl2)

sudo ./svc.sh install
sudo ./svc.sh start
sudo ./svc.sh status    # active (running) 확인
```

- 레포 단위 등록만 가능 (개인 계정 기준) → 레포마다 별도 폴더에 각각 설치
- 라벨은 `repo → Settings → Actions → Runners → 해당 Runner`에서 실제 값 확인 가능

## 4. 워크플로 파일 (`.github/workflows/deploy.yml`)

```yaml
name: Build and Deploy {서비스명}

on:
  push:
    branches: [main]
    paths:
      - 'backend/**'
      - 'Dockerfile.main'
      - 'docker-compose.yml'
      - '.github/workflows/deploy.yml'   # 워크플로 파일 자신도 반드시 포함

jobs:
  build-and-push:
    runs-on: [self-hosted, gpu, wsl2]    # 실제 등록된 라벨과 정확히 일치해야 함
    steps:
      - uses: actions/checkout@v4

      - name: Set image tag
        id: vars
        run: echo "tag=$(git rev-parse --short HEAD)" >> $GITHUB_OUTPUT

      - name: Build image
        run: |
          docker build -f Dockerfile.main \
            -t {레지스트리주소}/{이미지명}:${{ steps.vars.outputs.tag }} \
            -t {레지스트리주소}/{이미지명}:latest .

      - name: Push image
        run: |
          docker push {레지스트리주소}/{이미지명}:${{ steps.vars.outputs.tag }}
          docker push {레지스트리주소}/{이미지명}:latest

      - name: Deploy
        run: |
          export IMAGE_TAG=${{ steps.vars.outputs.tag }}
          cd {실제_운영_프로젝트_경로}     # Runner의 임시 checkout 경로가 아닌, 실제 운영 경로로 이동
          docker compose pull
          docker compose up -d --force-recreate {서비스명}
```

## 5. docker-compose.yml (레지스트리 pull 전용, 운영서버용)

```yaml
services:
  {서비스명}:
    image: {레지스트리주소}/{이미지명}:${IMAGE_TAG:-latest}
    # build: 항목 불필요 — 운영서버는 순수 pull만
    container_name: {서비스명}
    restart: always
    volumes:
      - ./ai_model:/workspace/ai_model
      - ./secrets/secrets.prod.json:/workspace/backend/backend/secrets/secrets.json
    deploy:
      resources:
        reservations:
          devices:
          - driver: nvidia
            capabilities: [gpu]
```

## 6. 운영서버에서 수동 pull (참고용)

```powershell
# insecure-registries 등록 (Docker Desktop → Settings → Docker Engine)
# { "insecure-registries": ["{레지스트리주소}"] }

cd {docker-compose.yml 위치}
$env:IMAGE_TAG = "latest"    # 또는 특정 커밋 해시
docker compose pull
docker compose up -d --force-recreate {서비스명}

docker ps | Select-String {서비스명}
docker inspect {컨테이너명} --format "{{.Config.Image}}"
```
