# WSL 디스크에 Docker Desktop에 MySQL 설치하기

앞선 포스팅에서 소스 파일을 WSL 파일 시스템으로 옮겨야 하는 이유를 다뤘다.
MySQL 역시 같은 이유로 **데이터 파일을 WSL 파일 시스템 안에 두는 것**이 중요하다.


## 왜 MySQL 데이터를 WSL 디스크에 두어야 하는가

MySQL은 데이터를 읽고 쓸 때 수많은 소규모 파일 I/O를 발생시킨다.
InnoDB 데이터 파일, 트랜잭션 로그, 바이너리 로그 등이 끊임없이 읽히고 쓰인다.

MySQL 데이터 디렉토리가 Windows 파일 시스템(`C:\`, `D:\`)에 있으면, Docker 컨테이너가 접근할 때마다 **Windows ↔ WSL 간 파일 시스템 변환**이 발생한다.
이는 소스 파일보다 훨씬 빈번한 I/O가 발생하는 DB 특성상 성능에 치명적이다.

또한 MySQL은 데이터 디렉토리에 대한 **Linux 파일 권한**을 엄격하게 요구한다.
Windows 파일 시스템에서는 이 권한이 제대로 적용되지 않아 컨테이너 기동 자체가 실패하는 경우가 많다.

따라서 MySQL의 data와 logs 디렉토리는 반드시 **WSL 파일 시스템 안**에 위치해야 한다.


## data / logs 폴더 위치 결정

Volume Mount로 MySQL 데이터를 관리할 때 폴더 위치를 어디에 둘지 고민이 생긴다.

### 방법 1. docker-compose.yml 과 같은 위치 (상대 경로)

```
~/mysql/
├── docker-compose.yml
├── data/
└── logs/
```

- 설정과 데이터가 한 곳에 모여 있어 찾기 편하다.
- 프로젝트 디렉토리를 삭제하면 데이터도 함께 사라진다.

### 방법 2. 별도 디렉토리 (절대 경로) — 권장

```
~/mysql/
└── docker-compose.yml

~/database/mysql/8.0/
├── data/
└── logs/
```

- 설정(docker-compose.yml)과 데이터를 분리하여 관리한다.
- 프로젝트 디렉토리를 삭제하거나 재구성해도 데이터가 유지된다.
- `/database/{DB종류}/{버전}/` 구조로 두면 다음과 같은 장점이 생긴다.
  - MySQL 외 PostgreSQL, Redis 등 다른 DB도 동일한 규칙으로 관리할 수 있다.
  - 버전별로 데이터가 분리되어 있어, MySQL 버전 업그레이드 시 기존 데이터를 보존한 채 새 버전을 병렬로 운영할 수 있다.

```
~/database/
├── mysql/
│   ├── 8.0/
│   │   ├── data/
│   │   └── logs/
│   └── 9.0/        ← 버전 업그레이드 시 추가
│       ├── data/
│       └── logs/
└── postgres/
    └── 16/
        ├── data/
        └── logs/
```

> MySQL 데이터는 장기간 쌓이는 자산이다.
> 실수로 프로젝트 디렉토리를 지워도 데이터가 남을 수 있도록 **별도 디렉토리(방법 2)**를 권장한다.
{: .prompt-tip}


## 디렉토리 생성

WSL 터미널에서 data와 logs 디렉토리를 미리 생성한다.
MySQL 컨테이너는 기동 시 데이터 디렉토리가 비어 있어야 초기화가 진행된다.

```bash
mkdir -p ~/database/mysql/8.0/data
mkdir -p ~/database/mysql/8.0/logs
```
{: .nolineno }

> `/database` 를 루트(`/`) 아래에 두려면 `sudo`가 필요하다. 개인 개발 환경에서는 홈 디렉토리 아래(`~/database/`)에 두는 것이 권한 문제 없이 간단하다.
{: .prompt-tip}


## docker-compose.yml 작성

```yaml
name: mysql-8.4
services:
  mysql:
    build: .
    #image: mysql:8.4
    container_name: mysql-8.4
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      TZ: Asia/Seoul
    
    volumes:
      - ~/database/mysql/8.4/data/data:/var/lib/mysql
      - ~/database/mysql/8.4/logs:/var/log/mysql
    
    ports:
      - "3306:3306"
    
    networks:
      - mysql-net

networks:
  mysql-net:
    external: true
```
{: .nolineno }

> volumes 경로는 WSL 절대 경로(`/home/<사용자명>/...`)로 지정해야 한다.
> `./data` 와 같은 상대 경로를 사용하면 docker-compose.yml 이 Windows 파일 시스템에 있을 경우 앞서 언급한 문제가 그대로 발생한다.
{: .prompt-warning}


## docker compose up 하기 전!

`docker-compose.yml`에서 정의한 `mysql-net`은 `external: true`로 선언되어 있기 때문에,
Docker가 자동으로 생성하지 않는다. 컨테이너를 올리기 전에 먼저 네트워크를 생성해야 한다.

```bash
docker network create mysql-net
```
{: .nolineno }

> 이 명령어를 실행하지 않으면 `docker compose up` 시 네트워크를 찾을 수 없다는 오류가 발생한다.
{: .prompt-warning}


## 컨테이너 실행

```bash
docker compose up -d
```
{: .nolineno }

정상 기동 여부는 아래 명령어로 확인한다.

```bash
docker ps
docker logs mysql
```
{: .nolineno }
